DeepBook Predictを理解するための図解メモ

この資料は、DeepBook本体の基礎を読んだチーム向けの続編です。ここでは、専門用語をなるべく避け、DeepBook Predictを「価格リスクを扱う部品」として理解します。

この資料で使う言い方：「オプション取引」という言葉は、チーム内で誤解を生みやすいため、ここでは 価格リスク商品 と呼びます。意味は「未来の価格がある条件を満たした時に、支払いが発生する仕組み」です。

1. 価格リスク商品の基礎

価格リスク商品 = 「未来の価格がどうなったら、誰が誰にいくら払うか」を決めるチケット

通常の売買は「今、SUIを買う」「今、SUIを売る」です。価格リスク商品は少し違います。未来のある時点で、価格が条件を満たしたら支払いが発生します。

現在 SUI = 1.00ドル。ユーザーは10,000 SUIを持っている。
→
条件を決める 30日後にSUIが0.80ドル未満なら支払いが出る。
→
満期で判定 条件成立なら受け取り。成立しなければ受け取りなし。

図解A：下落保護チケット

これは「SUIが下がる方に賭ける」ではなく、SUIを持っている人が、下落時のダメージを一部減らすために買うチケットです。

30日後にSUIが0.80ドル未満なら2,000 USDC受け取る 安心料：520 USDCの例
0.80ドル未満：受け取りあり 0.80ドル：判定ライン 0.80ドル以上：受け取りなし

数値例：SUIが1ドルの時に下落保護を買う

前提
現在価格1 SUI = 1.00 USDC
保有量10,000 SUI
現在価値10,000 USDC
守りたい期間30日
買うチケット0.80ドル未満なら2,000 USDC受取
仮の安心料520 USDC
結果の読み方

価格が下がらなければ、520 USDCは戻らない可能性があります。これは損ではなく、何も起きなかった時の安心料です。

価格が大きく下がった場合は、SUI本体の価値低下を、受け取り額で一部相殺します。

30日後のSUI価格SUI本体の価値チケット受取安心料最終的な価値意味
1.10ドル11,0000-52010,480SUIは上がった。安心料だけ払った状態。
1.00ドル10,0000-5209,480価格は変わらない。安心料分だけ減る。
0.75ドル7,5002,000-5208,980下落したが、受け取りでダメージを減らす。
0.60ドル6,0002,000-5207,480深く下落。完全には守れないが、一部補助される。

図解B：レンジチケット

レンジチケットは「30日後に価格がこの範囲に入っていたら支払い」という商品です。単なる上下予想ではなく、価格が大きく動かない場面や、Vault側のリスク設計に使えます。

30日後にSUIが0.90〜1.10ドルなら支払い 範囲に入るかを見る商品

2. DeepBook Predictで使われる概念

Predictは「価格リスク商品を作る部品」。商品を買う人、支払い金庫、価格判定の仕組みがセットになっている。
Oracle 判定に使う価格情報。例：SUI価格、BTC価格。
→
商品条件 期限、判定ライン、方向、範囲を指定する。
→
ユーザー口座 買ったチケット、残高、精算結果を管理する。
→
Vault / PLP 支払い原資の金庫。PLPは金庫に資金を入れる側。

部品を人間の言葉に置き換える

Vault / PLPは何をしているのか

保護を買う人と、支払い金庫の関係 ギャンブルではなく、リスクの引き受け

PLPの位置づけ：CLOB参加者とPredict参加者を並べて見る

DeepBook本体はCLOB、つまり板取引エンジンです。そこでは、買いたい人・売りたい人・板に流動性を置く人がいます。一方、DeepBook Predictでは、価格リスク商品を買う人と、その支払い原資を出すVault / PLPがいます。

DeepBook本体とPredict側の参加者 PLPは「Predict側の支払い金庫に資金を入れる人」

PLPを一言で言うと

役割

支払い金庫に資金を入れる人

保護チケットを買った人に支払うための原資をVaultに供給する。

得るもの

安心料の一部

保護を買うユーザーが払う安心料が、PLP側の収益源になる。

背負うもの

急変時の支払いリスク

価格が大きく動いて条件成立が増えると、Vaultから支払いが発生し、PLP側の価値が減る。

CLOBのMakerと、PredictのPLPは似ているが同じではない

比較CLOBのMakerPredictのPLP
場所DeepBook本体の注文板DeepBook PredictのVault
何を出すか買い注文・売り注文支払い原資となる資金
収益源スプレッド、手数料優遇、インセンティブなど保護を買う人が払う安心料
主なリスク不利な価格で約定する、在庫を抱える価格急変で支払いが増える
イメージ市場に売買の棚を並べる人保険金を払う金庫に資金を入れる人
重要：DeepBook Predictの理解で一番大事なのは、「買う人」と「金庫側」を分けることです。買う人は保護を得る。金庫側は安心料を得る代わりに、支払いリスクを背負います。PLPはこの金庫側の参加者です。

3. 具体的な活用例

目的は「価格を当てる」ではなく、Sui上で活動する人の資金を扱いやすくすること。

活用例A：ステーキング中の下落保護

実用シーン

ユーザーは10,000 SUIをステーキング中。今すぐ売れないが、30日以内の下落が怖い。

現在価格1.00 USDC
保有10,000 SUI
買う保護0.80ドル未満で2,000 USDC受取
仮コスト520 USDC
人間の言葉で説明

これは「下落に賭ける」のではなく、売れない期間のシートベルトです。価格が下がらなければ費用は戻らない可能性がありますが、暴落時のダメージを一部減らせます。

活用例B：TGE後の売却・保有・保護の分け方

TGE後にトークンを受け取ったチームは、すぐ全部売るとコミュニティからの印象が悪く、全部持つと暴落が怖い。この時、DeepBookを使うと「一部を売る」「一部を持つ」「一部に保護をつける」という3分割ができます。

受け取り 100,000 token。現在価格は1 token = 1 USDC。
→
20%を確保 20,000 tokenをDeepBook SpotでUSDC化。20,000 USDCを確保。
→
80%を保有 80,000 tokenは持ち続ける。上がれば利益を残せる。
→
暴落だけ保護 0.70ドル未満なら30,000 USDC受け取る保護を買う。仮コスト12,000 USDC。
TGE後の3分割イメージ 全部売りでも、全部保有でもない

活用例Bの計算を文章で読む

価格が1.50ドルに上がった

残した80,000 tokenは120,000 USDC相当。先に確保した20,000 USDCもある。保護費用12,000を引いても、合計128,000 USDC相当。

価格が1.00ドルのまま

残した80,000 tokenは80,000 USDC相当。先に確保した20,000 USDCを足して100,000。保護費用12,000を引くと88,000。

価格が0.60ドルに下がった

残した80,000 tokenは48,000 USDC相当。先に確保した20,000と、保護の受取30,000を足す。保護費用12,000を引いて86,000。

読み方：この方法は利益最大化ではありません。TGE直後に「全部売る」「全部持つ」の二択を避け、資金を残しながら時間を稼ぐ設計です。

4. 新たな戦略の考え方

ここでは専門戦略名を使わない。見るべきは「誰が安心料を払い、誰が受け取り、どこで損をするか」。
切り分け：保護を買うだけの設計は、資金防衛であって収益戦略ではありません。収益戦略として考えるなら、安心料を受け取る側、支払い金庫側、板を使ってリスクを調整する側を見る必要があります。

戦略1：安心料を受け取る金庫運用

ユーザーが下落保護を買うと、安心料を払います。その安心料を受け取るのがVault / PLP側です。普段は安心料が収益になりますが、価格が大きく動くと支払いが発生します。

金庫側の損益イメージ 保護を買う人とは逆の形になる
平常時

多くの保護が発動しなければ、金庫側は安心料を受け取って収益を得る。

急変時

SUIが判定ラインを下回るなど、条件成立が多発すると金庫側は支払いを負う。

戦略2：危ない金庫には入らない

金庫に資金を入れる側は、表面上の収益だけで判断してはいけません。「いつ、どの価格ラインで、どれくらい支払いが発生しそうか」を見る必要があります。

Vault残高 金庫にどれだけ資金があるか。
＋
最大支払い 最悪時にどれだけ払う可能性があるか。
＋
満期集中 支払いリスクが近い日に集中していないか。
→
判断 入る、待つ、抜ける、保護を追加する。
入りやすい状態

Vault残高が厚い。最大支払いが小さい。満期が分散している。受け取れる安心料が十分にある。

避けたい状態

明日満期の商品に支払いが集中している。特定の価格ラインを割ると一気に支払いが増える。表面収益は高いが一撃損失が大きい。

戦略3：板を見ながら支払いリスクを小さくする

DeepBook本体は板取引エンジンです。Predict側で価格リスクを引き受ける場合、DeepBook Spot / Marginで少しずつ反対売買をして、金庫側の危険を小さくできます。

価格が判定ラインに近づくほど、ヘッジを増やす 板取引エンジンがあるからできる調整
注意：これは高度な戦略です。メンバー向けには「Predictで引き受けたリスクを、DeepBookの板で少しずつ調整する」と理解すれば十分です。

戦略4：売る・守る・金庫側に回る、どれが有利か比べる

同じSUI価格リスクでも、取れる行動は1つではありません。DeepBookでは、Spotで売る、Predictで守る、Vault側で安心料を受け取る、といった選択肢を比較できます。

Spotで売る 価格を確定する。上昇余地は失う。
or
Predictで守る 安心料を払う。下落時に受け取る。
or
Vault側に入る 安心料を受け取る。急変時に支払いを負う。
→
比較する どの選択肢が一番リスクに見合うかを見る。

戦略5：範囲の外側だけを警戒する

多くの人が「上がるか下がるか」だけを考えます。しかし、実際には「この範囲に収まるなら問題ないが、外に出たら困る」という資金管理もあります。

範囲の外側だけを警戒する考え方 0.90〜1.10なら問題なし、外なら対策
戦略としての本質：DeepBookで見るべきものは「予想」ではなく、価格リスクを誰が持ち、誰が安心料を受け取り、どの時点で危険になるか。
注意：本資料の数値例は理解用の仮定であり、実際の価格・費用・収益は市場、満期、ボラティリティ、Vault状態、手数料、流動性によって変動する。
