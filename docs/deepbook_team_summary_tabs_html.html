Team Briefing / Sui DeepBook

DeepBookは「予測市場」ではなく、まず板取引エンジンとして理解する

DeepBook本体、Margin、Predictが同じブランドで語られるため混乱しやすい。まずはDeepBook本体を「Sui上の共通注文板」として理解し、その周辺にMarginやPredictが乗る、と整理する。

DeepBook 本体 Sui上の共通板取引エンジン。買い注文と売り注文を突き合わせる。
DeepBook Margin DeepBookの取引基盤を使った、レバレッジ・証拠金取引側の拡張。
DeepBook Predict 価格・期限・判定ラインを使った、予測/ヘッジ/オプション的商品の拡張。

1. 全体像

DeepBook = Sui上で複数のアプリが共通利用できる「板取引エンジン」

DeepBookをいきなりPredictから理解しようとすると混乱する。順番としては、まずDeepBook本体があり、その周辺にMarginやPredictのような金融商品・機能が乗る。

DeepBook 本体
Sui上の注文板。売買注文を並べ、条件が合えば取引を成立させる。
→
Margin
証拠金・レバレッジ取引など、取引機能を高度化する拡張。
→
Predict
価格・期限・ストライクを使い、ヘッジやオプション的商品を作る拡張。
まず覚える一文：DeepBook本体は「Suiの中にある共通の取引所エンジン」。

2. Uniswap型とDeepBook型の違い

Uniswapは「プールに向かって売買する」。DeepBookは「買いたい人と売りたい人の注文を突き合わせる」。この違いが最重要。

Uniswap型

誰でも使える両替プール

ユーザーは流動性プールに対してトークンを交換する。価格はプール内の残高比率で決まる。

イメージ：自動両替機

DeepBook型

誰でも使える板取引エンジン

ユーザーは板に並んだ買い注文・売り注文と取引する。価格は注文同士のマッチで決まる。

イメージ：Binanceの板をオンチェーン化

比較UniswapDeepBook
方式AMM / 流動性プールCLOB / 注文板
価格の決まり方プール残高の比率買い注文と売り注文の一致
強みシンプルで流動性提供しやすい指値・板の厚み・プロ向け取引に強い
成功の意味DeFiに共通両替プールを作ったSuiに共通板取引エンジンを作ろうとしている

3. DeepBookを構成する概念

DeepBookを理解するために、最初に押さえる概念はこの程度で十分。

概念意味たとえ

Bid買い注文「この価格なら買う」

Maker板に注文を置く人流動性を作る側
Taker板にある注文を取る人流動性を使う側

4. 他チェーン・他概念の成功例

DeepBookを理解するには、同じものを探すより「近い成功パターン」を見る方がわかりやすい。

例1

Uniswap

DeepBookとは方式が違うが、「共通取引インフラ」として最も重要な成功例。Uniswapは両替プール、DeepBookは板取引エンジン。

比較対象としては必須。ただし、DeepBookと同じ構造ではない。

例2

dYdX

分散型デリバティブ取引所。板とマッチングエンジンを使って、オンチェーン/分散型の高機能取引所を実現する方向の代表例。

DeepBook理解への示唆：DeFiでも「板取引」は成立しうる。

例3

Hyperliquid

オンチェーン注文板型取引所の成功例。CEXに近い使用感と、オンチェーン透明性を両立しようとしている。

DeepBook理解への示唆：高速チェーンなら板取引インフラは強力な武器になる。

例4

Derive / Aevo

オプション、perps、structured productsを扱うオンチェーン金融商品の先行例。Predictを理解する時の参考になる。

DeepBook理解への示唆：Predictは社会イベント予測ではなく、価格ヘッジ・オプション商品に近い。

5. 具体的活用例

DeepBookは、ユーザーから見ると「取引所そのもの」ではなく、ゲーム・支払い・ヘッジアプリの裏側で使われる共通エンジンになる。

例1

ゲーム内マーケット

Suiのゲームで、ゲーム内tokenをUSDCと交換したい。

ゲーム側が独自交換所を作ると大変。

DeepBookのpoolを使えば、ゲーム内から直接取引させられる。

例2

支払いアプリ

ユーザーはSUIしか持っていない。

店側はUSDCで受け取りたい。

支払いアプリが裏でDeepBookを使ってSUI → USDCに交換し、店にUSDCを渡す。

例3

Predict / ヘッジ商品

SUI保有者が「30日だけ下落に備えたい」。

アプリはDeepBook Predictのoracle / expiry / strike / binary positionを使って、DOWN側の商品を買わせる。

重要：これは社会イベント予想ではなく、価格リスク管理。

DeepBookの価値は、各アプリが自前で取引所を作らず、共通の流動性と板取引を使えること。

6. DeepBook Predictの位置づけ

PredictはPolymarket型ではなく、価格・期限・ストライクを使ったヘッジ/オプション的商品の拡張として見る

DeepBook Predictを「予測市場」とだけ見ると、ニュースや選挙の賭けを連想しやすい。しかしDeepBook文脈では、価格オラクル、期限、判定ラインを使って、価格リスクに対する商品を作る仕組みと見る方が自然。

要素意味活用イメージ
Oracle判定に使う価格情報SUI価格、BTC価格など
Expiry満期・判定タイミング7日後、30日後など
Strike上か下かを判定する価格ラインSUIが$Xを下回るか
Binary Position条件成立なら支払いが出るポジション下落保険、短期ヘッジ
Vertical Range価格が一定範囲に入るかを扱う商品レンジ相場向け商品
Vault / PLP流動性提供側の資金と持分LP収益・リスク管理
チーム向け一文：DeepBook Predictは「価格を当てる賭け」ではなく、「Sui上の価格リスクを商品化する部品」。
DeepBook理解用メモ：本体は注文板エンジン、Margin/Predictはその周辺に乗る金融機能として整理する。
